## Purpose
Magic Box highlights content that is being emphasized or **actively updated by AI/agents**. It provides visual feedback so users can see where intelligent systems are processing, modifying, or enhancing content.

## Rules
- **MUST** use the Living Design Magic Box component.
- **MUST** use Magic Box to indicate **active** AI/agentic generation or updates.
- **MUST** remove/hide Magic Box when the AI operation is complete (do not leave it on static content).
- **MUST** apply Magic Box to a **container/section** that wraps AI-driven content (not to individual leaf components).
- **MUST** place Magic Box on default color surfaces.
- **MUST NOT** use Magic Box on surfaces with custom colors.
- **MUST NOT** overuse Magic Box; excessive use reduces emphasis and creates visual clutter.
- **MUST NOT** use Magic Box when AI interaction is a core, always-on feature that does not need special emphasis.

## Usage
Use Magic Box to draw attention to elements that are being processed, modified, or enhanced by AI systems. Common contexts include:
- Content cards being generated by AI
- Containers with AI-recommended items
- Sections being updated by intelligent agents
- Components receiving real-time AI enhancements

## Variants
### Theme response
Magic Box colors respond to the “super agent” theme (Sparky, Squiggly, Marty, or Wibey). Use the component defaults for theme styling.

## States
- Active (AI operation in progress)
- Completed (Magic Box removed/hidden after completion)

## Accessibility
- Magic Box **MUST NOT** be the only way to communicate critical status. Pair it with accessible status text if users need to understand progress or completion.
- If you announce progress/completion, use appropriate live-region/status patterns as defined by your UI framework (and avoid overly chatty announcements).

## Token usage
- Prefer component defaults (Magic Box should be token-wired for color and effects tied to the agent theme).
- Do not restyle Magic Box via ad-hoc tokens or custom CSS.

## React usage (example)

```tsx
import { MagicBox } from "@/components/ui/MagicBox";

export function Example({ isUpdating }: { isUpdating: boolean }) {
  return (
    <MagicBox active={isUpdating}>
      {/* AI-driven content goes here */}
    </MagicBox>
  );
}
```

## Best practices
### Use when
- Use when indicating content that is being generated or updated by AI.
- Use when showcasing AI-enhanced or AI-recommended content.
- Use when providing visual feedback during agentic AI operations.

### Don’t use when
- Don't use when the content is static and no longer being actively updated by AI.
- Don't use when overuse would diminish the impact and create visual clutter.
- Don't use when the AI interaction is a core, expected feature rather than something to highlight.

## Do / Don’t
### Do
- Do use Magic Box on a container or section that wraps AI-driven content.
- Do use Magic Box on default color surfaces.

### Don’t
- Don't apply Magic Box to individual components.
- Don't use Magic Box on surfaces with custom colors.
